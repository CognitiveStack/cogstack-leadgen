## Commands

```bash
# Install dependencies
uv sync

# Run main entry point
uv run python main.py

# One-time: create Notion databases (requires NOTION_API_KEY and NOTION_PAGE_ID in .env)
uv run create_notion_databases.py

# Test the n8n webhook with 3 sample leads
export WEBHOOK_URL="https://n8n.bigtorig.com/webhook/lead-ingestion-v2"
export WEBHOOK_TOKEN="<token from .env>"
uv run test_webhook.py
```

There is no test suite yet. Syntax-check Python files with:
```bash

```

## Architecture

This pipeline serves **two call centres** — B2B (fleet operators) and B2C (individual consumers) — via parallel ingestion pipelines into separate Notion databases.

### B2B Pipeline

AI-powered B2B lead generation for fleet operator prospects. Goal: 10 qualified leads/month.

**Data flow:**
```
OpenClaw (Raspberry Pi 4, Tailscale VPN)
    │  POST JSON: { batch_id, leads[] }
    ▼
n8n Webhook (https://n8n.bigtorig.com/webhook/lead-ingestion-v2)
    │  Code node: validate → dedup by company name → create Notion records
    ▼
Notion Workspace: 3 linked databases
    ├── Leads DB      — 28-field schema; new leads arrive as "Pending QA"
    ├── Batches DB    — one record per ingestion run (status, counts, errors)
    └── Sources DB    — reference data (CIPC, eTenders, Yellow Pages, etc.)
    ▼
Claire reviews leads in Notion UI
    ├── QA Approved → Sent to Paul's B2B call centre
    └── QA Rejected → Feeds back into AI tuning
```

**n8n integration — two workflow versions:**
- `n8n_lead_ingestion_workflow.json` — v1 (8 Notion nodes). Had property-mapping issues. Keep for reference but do not use.
- `n8n_code_node.js` — v2 replacement. Single Code node using direct Notion REST API calls via `$http.request()` (n8n-native). This is the active implementation.

**B2B Scoring model (Composite Score):**
```
(Fleet Likelihood × 0.4) + (Tracking Need Score × 0.4) + (Fleet Size Bonus × 0.2)
```
- Fleet Likelihood / Tracking Need: 0–10 scale
- Fleet Size Bonus: 0 (small), 1 (medium), 2 (large)
- Phase B (future): auto-approve ≥ 7, auto-reject < 4

### B2C Pipeline

Consumer lead generation targeting individuals who have expressed verifiable interest in personal vehicle tracking. Sources: Hellopeter (competitor churn), Gumtree Wanted ads, Facebook public groups, MyBroadband forums, OLX, Reddit.

**Data flow:**
```
Hugo/OpenClaw (Raspberry Pi 4, Tailscale VPN)
    │  POST JSON: { batch_id, segment: "B2C", leads[] }
    ▼
n8n Webhook (https://n8n.bigtorig.com/webhook/b2c-lead-ingestion)
    │  Code node: validate → dedup by intent_source_url → create Notion records
    ▼
Notion Workspace: 2 new linked databases + shared Sources DB
    ├── B2C Leads DB  — consumer schema (phone, email, intent signal, dispute tracking)
    ├── B2C Batches DB — one record per ingestion run
    └── Sources DB    — shared with B2B; includes 6 B2C-specific sources
    ▼
Claire reviews B2C leads in Notion UI
    ├── QA Approved → Sent to B2C call centre
    └── QA Rejected → logged with rejection reason
```

**B2B Scoring model (B2C Composite Score):**
```
(Intent Strength × 0.6) + (Urgency Score × 0.4)
```
- Intent Strength: 0–10 (how explicitly did they ask for tracking?)
- Urgency Score: 0–10 (how recent/urgent is the need?)
- Phase B (future): auto-approve ≥ 7, auto-reject < 4

**B2C Deduplication:** keyed on `intent_source_url`. Same post URL = duplicate. Same person posting again on a different URL = fresh lead (re-qualified by new intent signal).

**Dispute tracking fields:** `Date Delivered`, `Rejection Deadline` (Notion formula: +5 days), `Dispute Status` (Pending / Accepted / Rejected by Client / Disputed).

**QA status states (both B2B and B2C):**
`Pending QA → QA Approved → Sent to Call Centre → Contacted → Converted / Not Interested`
`Pending QA → QA Rejected`
`Pending QA → Duplicate`

## Environment

Copy `.env` variables for local scripts. Required variables:
- `NOTION_API_KEY` — Notion integration token (`ntn_...`)
- `NOTION_PAGE_ID` — Parent page ID where databases live
- `LEADS_DB_ID`, `SOURCES_DB_ID`, `BATCHES_DB_ID` — generated by `create_notion_databases.py`, also in `notion_config.json`
- `B2C_LEADS_DB_ID`, `B2C_BATCHES_DB_ID` — generated by `create_b2c_database.py`, also in `notion_config.json`
- `B2C_WEBHOOK_URL` — `https://n8n.bigtorig.com/webhook/b2c-lead-ingestion`
- `B2C_WEBHOOK_TOKEN` — same bearer token as B2B, or a separate one configured in n8n
- `AGENT_BROWSER_EXECUTABLE_PATH=/usr/bin/google-chrome-stable` — tells agent-browser to use system Chrome instead of bundled browser

Notion API version in use: `2022-06-28` (stable). Formula and Rollup properties cannot be created via API — they must be added manually in the Notion UI after running the setup scripts (see `.claude/docs/SETUP_GUIDE.md`).
